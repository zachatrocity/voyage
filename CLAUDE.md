# Voyage — Developer Guide

Voyage is a self-hosted travel email aggregator built with **Dioxus 0.7.3** (frontend) and a **Go/Echo** backend that indexes Gmail via notmuch/mbsync. The frontend targets **Android** as the primary platform (CI builds a debug APK), with web and desktop as secondary targets.

---

## Stack

| Layer | Tech |
|---|---|
| Frontend | Dioxus 0.7.3, Rust |
| Styling | Tailwind CSS (auto-watched by `dx` in 0.7) |
| Icons | dioxus-free-icons 0.10 (lucide feature) |
| Backend | Go 1.23, Echo v4, notmuch CGo bindings |
| Mail sync | mbsync (isync) + notmuch |
| CI | GitHub Actions — prebuilt `dx` binary from GH releases |

---

## Project Structure

```
src/
  main.rs          # App root, router setup
  components/      # Reusable UI components
  views/           # Full page/screen components
assets/
  styling/         # CSS files
  tailwind.css     # Generated by dx (do not hand-edit)
Dioxus.toml        # App config, bundle ID: app.voyage.zachatrocity
Cargo.toml         # dioxus pinned to =0.7.3, dioxus-core pinned to =0.7.3
```

> ⚠️ **Important:** `dioxus` and `dioxus-core` are **pinned to `=0.7.3`** — do not bump to 0.7.4, it has a `try_recv` bug in `dioxus-core` that prevents compilation.

## Running Locally

```bash
# Web (dev server with hot reload)
dx serve --platform web

# Android (requires connected device/emulator + NDK)
dx build --platform android --target aarch64-linux-android
adb install -r target/dx/voyage/debug/android/app/*.apk
```

Tailwind is auto-compiled by `dx` in 0.7 if `tailwind.css` exists at root — no separate watcher needed.

## State Management

Dioxus 0.7 uses a **signals-first** reactive model. Key patterns:

### Local component state
```rust
let mut count = use_signal(|| 0);
// Read: count()
// Write: count.set(1) or count += 1
```
- Prefer `use_signal` over `use_hook` — signals track reads/writes automatically and trigger fine-grained re-renders.
- Never create `Signal::new()` manually (requires `.manually_drop()`). Always use `use_signal`.

### Shared state across components — Context
```rust
// In parent/root:
use_context_provider(|| Signal::new(AppState::default()));

// In any child:
let state = use_context::<Signal<AppState>>();
```
- Wrap state in a **newtype struct** so multiple different types can coexist in the context tree.
- Context flows **down only** — children read parent context, not vice versa.

### App-wide global state — GlobalSignal
```rust
static SEARCH_QUERY: GlobalSignal<String> = Signal::global(|| String::new());

// Anywhere in the app:
SEARCH_QUERY.set("flights".to_string());
let q = SEARCH_QUERY();
```
- Use `GlobalSignal` for truly app-wide state (current search query, auth status, selected email).
- Use context for state scoped to a subtree.
- Use local `use_signal` for component-internal state (loading flags, form fields).

### Recommended state shape for Voyage
```rust
// global state
static EMAILS: GlobalSignal<Vec<Email>> = Signal::global(|| vec![]);
static SEARCH_QUERY: GlobalSignal<String> = Signal::global(|| String::new());
static SELECTED_EMAIL: GlobalSignal<Option<String>> = Signal::global(|| None); // email ID

// per-component
let loading = use_signal(|| false);
let error = use_signal(|| Option::<String>::None);
```

## UI Components

### Official — DioxusLabs/components (shadcn-style)
The official component library uses a **shadcn copy-paste model** — components are added directly into your project source, not imported as a dependency:

```bash
dx components add button
dx components add input
dx components add card
```

This creates a `components/` folder with the Rust source + a `dx-components.css` link in your app root. Components are fully customizable since you own the code.

Browse available components: https://dioxuslabs.github.io/components/

**Recommended components for Voyage:**
- `card` — email list items
- `button` — actions (tag, archive)
- `input` — search bar
- `badge` — tags/labels on emails
- `scroll-area` — email list

### Icons
Already configured: `dioxus-free-icons` with `lucide` feature.

```rust
use dioxus_free_icons::icons::lucide_icons::LuSearch;
use dioxus_free_icons::Icon;

rsx! {
    Icon { icon: LuSearch, width: 24, height: 24 }
}
```

### Tailwind
Already set up. The `tailwind.css` is auto-compiled. Add classes directly in `class:` attributes:

```rust
rsx! {
    div { class: "flex flex-col gap-4 p-4 bg-background text-foreground",
        // ...
    }
}
```

Use `dx-components.css` theme variables (`bg-background`, `text-foreground`, `bg-card`, etc.) for consistent theming with the component library.

## API Integration

Backend runs at `http://localhost:8181` (or configured via env).

```
GET /api/v1/search?q=<query>     — search emails
GET /api/v1/email/:id            — get single email
POST /api/v1/email/:id/tags/:tag — add tag to email
GET /health                      — health check
```

For async API calls in Dioxus, use `use_resource` (replaces `use_future`):

```rust
let emails = use_resource(move || async move {
    reqwest::get(format!("http://localhost:8181/api/v1/search?q={}", SEARCH_QUERY()))
        .await?
        .json::<Vec<Email>>()
        .await
});

match &*emails.read() {
    Some(Ok(data)) => { /* render */ }
    Some(Err(e)) => { /* error */ }
    None => { /* loading */ }
}
```

## CI/CD

`.github/workflows/ci.yml` builds a debug APK on every push to `main`:
- Downloads prebuilt `dx` binary from GH releases (v0.7.3, x86_64-linux)
- Builds via `dx build --platform android --target aarch64-linux-android`
- Uploads APK artifact from `target/dx/voyage/debug/android/app/`

## Known Issues / Watch Out

- **dioxus 0.7.4 is broken** — pinned to `=0.7.3` in Cargo.toml intentionally. The `try_recv` bug in `dioxus-core` prevents compilation. Do not upgrade until fixed upstream.
- Backend port 8080 is taken by signal-cli on this machine — backend runs on **8181**.
- Mail sync is limited to `MaxMessages 500` from INBOX only to avoid downloading entire Gmail history.
